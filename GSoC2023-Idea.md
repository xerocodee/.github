**Contributors' Information**

These suggestions came from our engineers and users. They are occasionally ambiguous or incomplete. If you want to make a proposal based on these ideas, you should first contact the developers to learn more about the specific suggestion you're interested in.

Acceptance as a Google Summer of Code contributor is extremely competitive. Accepted contributors have often conducted extensive study on the technology of their proposed project and have maintained regular communication with possible mentors. It is **not enough to just copy and paste an idea here**. Creating a wholly fresh concept without first consulting possible mentors, on the other hand, seldom works.

### 2: Full-Stack & Web frontend focused

---

**Skills required/preferred:** Javascript, React.js, Typescript, REST API, Tailwind CSS, Webpack etc.

**Mentors:**

**Size of project:** 350 hours

**Difficulty:** Medium to hard ( depending on chosen scope )

**Goals:** To create a user-friendly interface that is easy to use, even for users with no programming experience. Users should be able to drag and drop custom UI components to different sections of the design page, allowing them to create unique and custom designs. The website builder should support four different frameworks, such as React, Next, Vue and Typescript to export the design code to the user's preferred framework, so they can continue to work on their project outside of the website builder.
